## WHY THIS DOCUMENT EXISTS

Terminology drift is the silent failure mode of multi-product doc sets. Over time, "the station chief" starts meaning slightly different things in different docs, "signal" and "synthesis" blur, "Passage" gets used loosely. Each drift on its own is small. The compound effect is that the vocabulary stops being load-bearing.

This file catches drift early by giving every shared term exactly one canonical definition. When a product doc uses a term, it inherits the meaning from here. When two docs disagree about a term, this file resolves the conflict.

---

## HOW THIS DOCUMENT IS USED

Every new session opening the OS codebase should be able to read this glossary in under five minutes and emerge with the full shared vocabulary. Every product doc set should assume the reader has read this glossary and not re-define terms already defined here.

When a new term is introduced in a product doc, the author should ask: is this term shared across products, or product-specific? If shared, add it here. If product-specific, define it inline in the product doc.
