# Atlas

**Role:** Decision capture. The product that makes the between-state visible — decisions forming, positions developing, work that was seeded by a Dispatch signal or an Explore image and hasn't resolved yet.

**Status:** On hold as of 2026-04-09.

**Location:** Atlas currently lives as a separate repository at `~/claude-projects/atlas/` with its own stack (Next.js + Supabase + Voyage AI for document ingestion and retrieval). This directory (`docs/atlas/`) reserves Atlas's place in the OS taxonomy. Its strategic doc set will be established when development resumes and will live here alongside the other product doc sets.

**Why Atlas matters:** See `../os/PASSAGE.md` — Atlas is "the decision layer. The product that makes Passage structurally legible. Without Atlas, Dispatch generates insights that evaporate and Explore offers nourishment that doesn't compound. Atlas is where the passage deposits — not endpoints, but sediment."

**When Atlas comes off hold:** Create the full canonical 14-file doc set here (MANDATE, SYSTEM-BRIEF, ARCHITECTURE, etc. — see `../os/DOC-AUTHORITY.md` § THE CANONICAL PRODUCT DOC SET) and decide whether Atlas should fold into the shared OS codebase as a white-label instance or remain as a separate repository.
